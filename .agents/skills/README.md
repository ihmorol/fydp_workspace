## Practical Usage

- Start with `problem-statement` before committing to a topic.
- Use `scientific-brainstorming` only while the idea space is still open.
- Switch to `hypothesis-generation` once you have observations, candidate variables, or a sharper question.
- Use `research-workflow` as the top-level orchestrator for literature work, then call the database-specific skills as needed.
- Prefer `scientific-writing` as the main writing skill; use `paper-writing` for simpler drafting support.
- Use `scientific-manuscript-review` and `scientific-clarity-checker` together before submission.
- Use `venue-templates` before final formatting, not after the PDF is already finished.
